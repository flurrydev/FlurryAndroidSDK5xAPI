Detailed Description

The Flurry Android Analytics Agent allows you to track the usage and behavior of your Android application on users' devices for viewing in the Flurry Analytics system.

Author
2009-2014 Flurry, Inc. All Rights Reserved

Set of methods that allow developers to capture detailed, aggregate information regarding the use of their app by end users.

Flurry Publisher Support Site for more information

static void onEndSession ( Context  context)
static

End a Flurry session for the given Context.

Since
1.0 Pop a session off the stack for this Activity. If no sessions remain after the call to onEndSession(Context) and no new calls to onStartSession(Context, String) are made within the timeout specified (which defaults to 10 seconds) then the session will truly end. Calls to onEndSession should be made from the Activity's onStop method in order to assure a complete analytics lifecycle for the Activity. Please note that if your app only runs on ICS+ (minSdkVersion >= 14) Activity-based sessions will be handled for you automatically. static void onStartSession ( Context  context)
static

Start or continue a Flurry session for the project on the given Context.

Since
1.0 So long as there is any Context that has called onStartSession(Context) but not onEndSession(Context), the session will be continued. Also, if a new Context calls onStartSession(Context) within 10 seconds (the default session timeout length) of the last Context calling onEndSession, then the session will be resumed, instead of a new session being created. Session length, usage frequency, events and errors will continue to be tracked as part of the same session. This ensures that as a user transitions from one Activity to another in your application they will not have a separate session tracked for each Activity, but will have a single session that spans many activities. onStartSession should be called from within the onStart method of the activity in question. The first call to onStartSession will set up the configuration for all later occurring calls to onStartSession. Please note that if your app only runs on ICS+ (minSdkVersion >= 14) Activity-based sessions will be handled for you automatically. There is no need to call this method unless you wish to track another type of Context.
// The Activity's onStart method
* @Override protected void onStart() { // Start the Flurry session
* FlurryAgent.onStartSession(this); }
*
Note
You MUST initialize the Flurry SDK via #Flurry.init(String) before calling this method. Also, it is important to match any call to onStartSession(Context) in the Android onStart method with a call to onEndSession(Context) in the Android onEnd method.
See Also
setContinueSessionMillis(long) for details on setting a custom session timeout.
Parameters
contextA reference to a android.content.Context object such as an android.app.Activity or an android.app.Service.

static void setContinueSessionMillis ( long  millis)
static

Set the timeout for expiring a Flurry session.

Since
1.0 The default value for a session timeout is 10,000 milliseconds. The minimum timeout for a session is 5,000 milliseconds. Calling onStartSession(Context, String) within the set time will cause the previous session to resume. Calling onStartSession(Context, String) after the set amount of time has passed will result in a new session being created.
Note
This must be called before onStartSession to have an effect.
FlurryAgent.setContinueSessionMillis(60 * 1000); // Set session timeout to 60 seconds
* FlurryAgent.onStartSession(this, kApiKey);
*
See Also
onStartSession(Context, String) and onEndSession(Context)
Parameters
millisThe time in milliseconds to set the session timeout to. Minimum value of 5000.
